### Wurzel
**`src/models/dixon_coles.py:96-100` + `src/config.py:110-111`:**

```python
WC2026_BOOST = 3.0
# In _prepare_arrays:
weights = weights * np.where(wc2026_mask, WC2026_BOOST, 1.0)
```

Berechnung des effektiven Gewichts:
- WM 2026 Match (1 Tag alt): `exp(-0.0065 × 1) × 1.0 × 3.0 = 2.98`
- Copa-2024-Match (1 Jahr alt): `exp(-0.0065 × 365) × 1.0 = 0.094`
- → WM-Match zählt **~32× mehr** als Copa-2024

Mit nur 2 WM-Matches (Mexico vs SA, Korea vs Czechia) und nur ~5 historischen Mexico-Matches mit signifikantem Restgewicht, dominiert das WM-Spiel komplett. Der Optimizer findet eine Lösung mit Mexico.defence=-4.04 (perfekter Fit für SA=0 Tore).

**`src/models/dixon_coles.py:201`** (kein Bounds-Constraint):
```python
bounds = [(None, None)] * (2 * n) + [(None, None), (-0.5, 0.0)]
```
Attack/Defence sind **unbounded** → Optimizer kann beliebig große Werte produzieren.

### Empfohlene Fixes
1. **`WC2026_BOOST` von 3.0 → 1.5** (oder dynamisch: höher bei mehr Spielen)
2. **Bounds einführen**: `attack/defence ∈ [-3.0, 3.0]`
3. **Optional:** Mexico/Argentina/Morocco-Werte separat sanity-checken nach jedem Retrain

## C. MITTEL — Multi-Klassen-Kalibrierung Normalisierungsrisiko

**File:** `src/ensemble/calibration.py:32-37`

Per-Klasse Isotonic + Row-Renormalize. Bei sehr kleinen Row-Sums (Edge-Cases an Trainings-Distributionsrand) numerisch instabil. Aktueller Guard: nur `row_sums == 0`.

**Fix:** Threshold `if row_sum < 1e-6: row = uniform_prior` statt division.

## D. MITTEL — xG-Live Dedup-Logik Timezone-Issue

**File:** `src/features/xg_live.py:49-50`

```python
df["_date_day"] = pd.to_datetime(df["date"]).dt.normalize()
```

Wenn StatsBomb tz-aware (UTC) und Sofascore naive (lokal): vergleicht inkompatible Datumsobjekte → mögliche Duplikate.

**Fix:** `pd.to_datetime(...).dt.tz_localize(None).dt.normalize()` vor Dedup.

## E. NIEDRIG — Sofascore xG Period-Auswahl
